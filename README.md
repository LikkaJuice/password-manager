# Password Manager (CLI, Python + SQLite + Fernet)

Небольшое CLI‑приложение на Python для безопасного хранения паролей.  
Пароли шифруются с помощью Fernet (`cryptography`), хранятся в SQLite, а доступ к ним защищён мастер‑паролем.[web:491][web:447]

## Возможности

- Мастер‑пароль при первом запуске, хранится только в виде SHA‑256 хэша.
- Проверка мастер‑пароля при каждом запуске перед выполнением команд.
- Хранение записей в формате:
  - **Название** (откуда, например, `Google`)
  - **Логин**
  - **Пароль** (зашифрованный текст)
- Шифрование/дешифрование паролей через Fernet.
- Ключ шифрования хранится локально в файле `.key` (создаётся автоматически при первом запуске).
- Все данные лежат в базе `passwords.db` (SQLite).
- CLI‑команды:
  - `add` — добавить новую запись
  - `get` — получить пароль по названию
  - `list` — показать список всех записей
  - `delete` — удалить запись по названию
  - `new` — сгенерировать новый пароль.[web:458][web:488]

## Структура проекта

Файлы в корне проекта:

- `db_manager.py`  
  Класс **DatabaseManager**:
  - создаёт и инициализирует базу данных `passwords.db`;
  - таблица `master_password` для хранения SHA‑256 хэша мастер‑пароля;
  - таблица `passwords` для хранения названия, логина и зашифрованного пароля;
  - методы:
    - `set_master_password`, `get_master_password`;
    - `add_password`, `update_password`, `get_password`, `list_passwords`, `delete_password`.[web:447]

- `encryption_manager.py`  
  Класс **EncryptionManager**:
  - работает с шифрованием на базе Fernet;
  - в `__init__` проверяет наличие файла ключа `.key`:
    - если файл есть — считывает ключ;
    - если файла нет — генерирует новый и сохраняет;
  - методы:
    - `encrypt(data: str) -> bytes`;
    - `decrypt(token: bytes) -> str`.[web:452][web:459]

- `password_manager.py`  
  Класс **PasswordGenerator**:
  - логика мастер‑пароля:
    - `setup_master_password()` — установка при первом запуске;
    - `authenticate()` — запрос и проверка мастер‑пароля;
  - генерация паролей:
    - `PasswordOptions` — длина и набор символов;
    - `generate_password(options)` — создание случайного пароля;
    - `generate_password_interactive()` — интерактивный режим;
  - операции с хранилищем:
    - `add_password(name, login, password, overwrite=False)`;
    - `get_password(name)`;
    - `list_passwords()`;
    - `delete_password(name)`;
  - `show_menu()` — простое текстовое меню (опционально).[web:456][web:458]

- `cli.py`  
  Входная точка и CLI‑парсер:
  - реализует команды `add`, `get`, `list`, `delete`, `new` через `argparse`;
  - при запуске:
    - создаёт `PasswordGenerator`;
    - при необходимости вызывает `setup_master_password`;
    - запрашивает мастер‑пароль через `authenticate`;
    - выполняет выбранную команду.[web:450][web:497]

## Требования

- Python 3.10+ (рекомендуется).
- Установленная библиотека `cryptography`:

```bash
pip install cryptography
```

SQLite входит в стандартную библиотеку Python, отдельной установки не требует.[web:431]

## Установка и запуск (macOS / Linux)

1. Склонировать репозиторий или скопировать файлы в папку:

   ```bash
   git clone <URL_РЕПОЗИТОРИЯ>
   cd password-manager
   ```

2. (Рекомендуется) создать виртуальное окружение:

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```

3. Установить зависимости:

   ```bash
   pip install cryptography
   ```

4. Первый запуск (создание БД, ключа и мастер‑пароля):

   ```bash
   python3 cli.py
   ```

   Приложение:
   - создаст `passwords.db` и `.key`, если их нет;
   - попросит ввести и подтвердить новый мастер‑пароль (ввод скрытый);
   - после этого покажет помощь по командам (`--help`).[web:491][web:496]

## Использование

Все команды запускаются через `cli.py`.

### Добавить новую запись

```bash
python3 cli.py add Google mylogin
```

Шаги:

1. Ввести мастер‑пароль.  
2. Ввести пароль для учётной записи `Google` (дважды).  
3. Программа зашифрует пароль и сохранит запись в базе.

Можно передать пароль сразу:

```bash
python3 cli.py add Google mylogin -p MySecret123! --overwrite
```

Флаг `--overwrite` позволяет обновить существующую запись с тем же названием.[web:482][web:488]

### Получить пароль по названию

```bash
python3 cli.py get Google
```

После ввода мастер‑пароля приложение выведет:

- название;
- логин;
- расшифрованный пароль.

### Показать список всех записей

```bash
python3 cli.py list
```

Показывает названия и логины всех сохранённых записей (без раскрытия паролей).

### Удалить запись

```bash
python3 cli.py delete Google
```

Удаляет запись с указанным названием.

### Сгенерировать новый пароль

```bash
python3 cli.py new
```

или с параметрами:

```bash
python3 cli.py new -l 24 --no-special
```

Параметры:

- `-l, --length` — длина пароля (по умолчанию 16);
- `--no-upper` — не использовать заглавные буквы;
- `--no-lower` — не использовать строчные;
- `--no-digits` — не использовать цифры;
- `--no-special` — не использовать спецсимволы.

Пароль выводится в консоль и **не** сохраняется в БД.[web:488]

## Безопасность и ограничения

- Мастер‑пароль хранится только в виде SHA‑256 хэша, исходное значение не сохраняется.
- Ключ шифрования хранится в `.key` рядом со скриптами:
  - **не** добавляйте этот файл в публичный репозиторий;
  - при потере `.key` расшифровать существующие пароли будет невозможно.
- Приложение работает локально в терминале, без сети и синхронизации.[web:458][web:492]
